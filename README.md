# Book Library Project

Проект библиотеки на Django для управления книгами, авторами и жанрами.

## Описание проекта

Веб-приложение для каталогизации книг с возможностью:

- Добавления авторов и книг
- Категоризации книг по жанрам
- Просмотра списка книг с фильтрацией по жанрам
- Администрирования через Django Admin

## Технологии

- Python 3.12
- Django 5.x
- SQLite (по умолчанию)

## Модели данных

### Author (Автор)

* `name` - имя автора
* `bio` - биография

### Genre (Жанр)

* `name` - название жанра
* `description` - описание жанра

### Book (Книга)

* `title` - название книги
* `author` - ForeignKey связь с Author
* `published_date` - дата публикации
* `description` - описание книги
* `genre` - ManyToManyField связь с Genre

## Ответы на вопросы (Задание 3)

#### Вопрос 1: Как изменился бы запрос, если бы мы использовали ForeignKey вместо ManyToManyField?

При использовании `ForeignKey` каждая
книга могла бы принадлежать только одному жанру. Это сделало бы
невозможным поиск книг, принадлежащих одновременно нескольким жанрам
(например, "Фантастика" И "Детектив"). Запрос количества книг в жанре
остался бы аналогичным, но структура данных стала бы менее гибкой.

#### Вопрос 2: Какие проблемы могут возникнуть при использовании ManyToManyField в запросах?

1. **Дублирование записей** - при фильтрации по нескольким жанрам одна книга может появиться несколько раз, требуется использовать `distinct()`
2. **Сложность поиска книг с несколькими жанрами** - обычный `filter()` с несколькими условиями не работает, нужно использовать цепочку `filter()` или Q-объекты
3. **Проблемы производительности** - возникает проблема N+1 запроса, требуется использовать `prefetch_related()` для оптимизации
4. **Усложнение агрегаций** - подсчет книг в жанре может давать неверные результаты без `distinct=True`
