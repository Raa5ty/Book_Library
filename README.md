# Book Library Project

---



**Проект библиотеки на Django для управления книгами, авторами, жанрами, издательствами, магазинами и отзывами.**

---



## Описание проекта

Веб-приложение для каталогизации книг с расширенной функциональностью:

- Управление авторами и книгами
- Категоризация книг по жанрам
- Привязка книг к издательствам
- Отслеживание наличия книг в магазинах
- Система отзывов и рейтингов
- Выполнение сложных аналитических запросов через Django ORM
- Оптимизированные страницы с результатами запросов

## Технологии

- Python 3.12
- Django 6.x
- SQLite (по умолчанию)

## Модели данных и связи

### Author (Автор)

| Поле | Тип    | Описание    |
| -------- | --------- | ------------------- |
| name     | CharField | Имя автора |
| bio      | TextField | Биография  |

### Genre (Жанр)

| Поле    | Тип    | Описание            |
| ----------- | --------- | --------------------------- |
| name        | CharField | Название жанра |
| description | TextField | Описание жанра |

### Publisher (Издательство)

| Поле | Тип    | Описание                          |
| -------- | --------- | ----------------------------------------- |
| name     | CharField | Название издательства |
| country  | CharField | Страна расположения     |

### Store (Магазин)

| Поле | Тип    | Описание                    |
| -------- | --------- | ----------------------------------- |
| name     | CharField | Название магазина   |
| city     | CharField | Город расположения |

### Review (Отзыв)

| Поле   | Тип                    | Описание                  |
| ---------- | ------------------------- | --------------------------------- |
| book       | ForeignKey                | Связь с книгой        |
| rating     | PositiveSmallIntegerField | Оценка (1-5)                |
| comment    | TextField                 | Текст комментария |
| created_at | DateTimeField             | Дата создания         |

### Book (Книга)

| Поле       | Тип          | Описание                                                      |
| -------------- | --------------- | --------------------------------------------------------------------- |
| title          | CharField       | Название книги                                           |
| author         | ForeignKey      | Связь с автором (один ко многим)             |
| publisher      | ForeignKey      | Связь с издательством (один ко многим) |
| published_date | DateField       | Дата публикации                                         |
| description    | TextField       | Описание книги                                           |
| genre          | ManyToManyField | Связь с жанрами (многие ко многим)         |
| stores         | ManyToManyField | Связь с магазинами (многие ко многим)   |

## Схема связей между моделями

```
┌─────────────┐ ┌─────────────┐ ┌─────────────┐
│ Author │ │ Publisher │ │ Genre │
│ 1 │ │ 1 │ │ N │
└──────┬──────┘ └──────┬──────┘ └──────┬──────┘
│ │ │
│ ForeignKey │ ForeignKey │ ManyToMany
↓ ↓ ↓
┌─────────────────────────────────────────────────────────┐
│ Book │
│ • author (ForeignKey → Author, 1:N) │
│ • publisher (ForeignKey → Publisher, 1:N) │
│ • genre (ManyToMany → Genre, N:M) │
│ • stores (ManyToMany → Store, N:M) │
└─────────────────────────────────────────────────────────┘
↑ ↑
│ ForeignKey │ ManyToMany (через Book)
│ │
┌──────┴──────┐ ┌──────┴──────┐
│ Review │ │ Store │
│ 1 │ │ N │
└─────────────┘ └─────────────┘
```

## Пояснение связей:

| Связь        | Тип                                      | Объяснение                                                                                                                                   |
| ----------------- | ------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Book → Publisher | ForeignKey (один ко многим)     | Одно издательство может выпустить много книг, но у книги только одно издательство |
| Book → Store     | ManyToMany (многие ко многим) | Книга может продаваться в нескольких магазинах, в магазине много книг                       |
| Book → Review    | ForeignKey (один ко многим)     | У книги может быть много отзывов, отзыв принадлежит одной книге                                   |

## Выполнение запросов через веб-интерфейс (Задание PRO)

Страница `/search/` отображает результаты 5 сложных запросов:

### Запрос 1: Книги издательств из России

**python**

```
Book.objects.filter(publisher__country='Россия').select_related('author', 'publisher')
```

**Результат:** Все книги, выпущенные издательствами из России (Эксмо, АСТ)

### Запрос 2: Книги, продающиеся в магазинах Москвы

**python**

```
Book.objects.filter(stores__city='Москва').prefetch_related('stores').distinct()
```

**Результат:** Все книги, доступные в магазинах Москвы

### Запрос 3: Книги со средней оценкой выше 4.5

**python**

```
Book.objects.annotate(avg_rating=Avg('reviews__rating')).filter(avg_rating__gt=4.5)
```

**Результат:** Книги с высокими пользовательскими оценками

### Запрос 4: Количество книг в каждом магазине

**python**

```
Store.objects.annotate(book_count=Count('books')).order_by('-book_count')
```

**Результат:** Рейтинг магазинов по количеству книг

### Запрос 5: Магазины с книгами после 2010 года

**python**

```
Store.objects.filter(books__published_date__year__gt=2010).annotate(book_count=Count('books')).distinct().order_by('-book_count')
```

**Результат:** Магазины, где продаются современные книги, отсортированные по ассортименту


## Оптимизация запросов

### Использование select_related()

`select_related()` используется для ForeignKey-связей (Author, Publisher). Он выполняет SQL JOIN и загружает связанные данные одним запросом:

```
# Без оптимизации - N+1 запросов
books = Book.objects.all()
for book in books:
    print(book.author.name)  # Дополнительный запрос для каждой книги

# С оптимизацией - 1 запрос
books = Book.objects.select_related('author', 'publisher').all()
for book in books:
    print(book.author.name)  # Данные уже загружены
```

### Использование prefetch_related()

`prefetch_related()` используется для ManyToMany-связей (Genre, Store, Review). Он выполняет отдельные запросы и объединяет результаты в Python:

```
# Без оптимизации - множество запросов
books = Book.objects.all()
for book in books:
    for genre in book.genre.all():  # Дополнительный запрос на каждую книгу
        print(genre.name)

# С оптимизацией - 2 запроса (книги + жанры)
books = Book.objects.prefetch_related('genre', 'stores', 'reviews').all()
for book in books:
    for genre in book.genre.all():  # Данные уже загружены
        print(genre.name)
```

### Агрегация с annotate()

`annotate()` позволяет выполнять вычисления на уровне БД:

```
# Вычисление среднего рейтинга прямо в SQL
Book.objects.annotate(avg_rating=Avg('reviews__rating'))

# Подсчет количества связанных объектов
Store.objects.annotate(book_count=Count('books'))
```

### Использование distinct()

Для ManyToMany-связей может потребоваться `distinct()` для устранения дублей:

```
# Без distinct() книга может появиться несколько раз
Book.objects.filter(stores__city='Москва')  # возможны дубли

# С distinct() - уникальные записи
Book.objects.filter(stores__city='Москва').distinct()
```


## Визуализация оптимизации на странице запросов

На странице `/search/` каждый запрос помечен бейджами оптимизации:

* 🟢 **select_related()** - для ForeignKey (author, publisher)
* 🔵 **prefetch_related()** - для ManyToMany (genre, stores, reviews)
* 🟡 **annotate() + Count/Avg** - для агрегации
* 🟣 **distinct()** - для устранения дублей

## Ответы на вопросы (Задание 3)

#### Вопрос 1: Как изменился бы запрос, если бы мы использовали ForeignKey вместо ManyToManyField?

При использовании `ForeignKey` каждая
книга могла бы принадлежать только одному жанру. Это сделало бы
невозможным поиск книг, принадлежащих одновременно нескольким жанрам
(например, "Фантастика" И "Детектив"). Запрос количества книг в жанре
остался бы аналогичным, но структура данных стала бы менее гибкой.

#### Вопрос 2: Какие проблемы могут возникнуть при использовании ManyToManyField в запросах?

1. **Дублирование записей** - при фильтрации по нескольким жанрам одна книга может появиться несколько раз, требуется использовать `distinct()`
2. **Сложность поиска книг с несколькими жанрами** - обычный `filter()` с несколькими условиями не работает, нужно использовать цепочку `filter()` или Q-объекты
3. **Проблемы производительности** - возникает проблема N+1 запроса, требуется использовать `prefetch_related()` для оптимизации
4. **Усложнение агрегаций** - подсчет книг в жанре может давать неверные результаты без `distinct=True`
